# Memory-Match-Game
รายงานสรุปโครงงาน: การประยุกต์ใช้ GitHub ในการทำงานร่วมกันอย่างมีระบบ <br>
วิชา: [ชื่อวิชาของหนู] | ปีการศึกษา: 2568 (ภาคเรียนที่ 2) กลุ่มที่: [ใส่หมายเลขกลุ่ม] | โปรเจกต์: เกมจับคู่ภาพ (Memory Match Game) <br>
1. วัตถุประสงค์ (Objectives)
เพื่อประยุกต์ใช้เครื่องมือ GitHub ในการบริหารจัดการซอร์สโค้ด (Source Code Management) และจำลองกระบวนการทำงานร่วมกันในรูปแบบทีมวิศวกรซอฟต์แวร์ โดยเน้นความถูกต้องของขั้นตอนการตรวจสอบโค้ดและการรวมงานอย่างเป็นระบบ <br><br>

2. ขั้นตอนการทำงานและเทคโนโลยีที่ใช้ (Workflow & Stack) <br>
ภาษาที่ใช้: HTML5, CSS3 และ JavaScript (Vanilla JS) <br>
• กระบวนการทำงาน (GitHub Workflow): <br>
• Branching Strategy: สมาชิกแยกกิ่ง (Branch) ตามส่วนงานที่ได้รับมอบหมาย เพื่อป้องกันการแก้ไขไฟล์ทับซ้อนกัน <br>
• Pull Request (PR): ใช้ระบบส่งคำขอรวมงานเพื่อให้หัวหน้าทีมตรวจสอบ (Code Review) ก่อนเข้าสู่กิ่งหลัก (Main) <br>
• GitHub Pages: ใช้สำหรับการจัดส่งซอฟต์แวร์ (Deployment) เพื่อให้สามารถใช้งานผ่านลิงก์เว็บไซต์ได้ทันที  <br><br>

3. การแบ่งหน้าที่ภายในทีม (Team Roles)<br>
หัวหน้าทีม (Leader/DevOps): จัดการโครงสร้าง Repository, ตรวจสอบ Pull Request และควบคุมการ Deploy <br>
สมาชิก (Developers): พัฒนาส่วนประกอบของเกม (Logic, UI, Styles) และส่งงานผ่านระบบ Branching <br><br>

4. สรุปผลการดำเนินงาน (Key Results) <br>
Collaborative Success: มีประวัติการทำงานชัดเจนผ่านระบบ Commits และ Contributors Graph แสดงให้เห็นถึงการมีส่วนร่วมของสมาชิกทั้ง 5 คนอย่างเท่าเทียม <br>
Quality Control: งานที่ผ่านการรวม (Merge) ทั้งหมดผ่านการตรวจสอบความขัดแย้งของโค้ด (Conflict Check) ทำให้โปรเจกต์มีความเสถียร <br>
Live Deployment: โปรเจกต์สามารถรันผลลัพธ์ได้จริงบนหน้าเว็บเพจ ทำให้สะดวกต่อการนำเสนอและการใช้งานของผู้ใช้ <br><br>
